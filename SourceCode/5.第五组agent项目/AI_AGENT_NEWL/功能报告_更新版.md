## 一、项目概述

本项目面向医学图像分割方向，构建了一个本地可运行的 AI Agent 系统。系统同时覆盖课程要求中的知识资产、评测基准和知识问答应用，并在 Part B 中实现了两个自选模块：

| 模块 | 难度等级 | 当前状态 |
|---|---:|---|
| 进阶动态编排 | Lv.3 | 已实现运行时计划生成、动态节点插入、retry、fallback recovery 和决策轨迹 |
| 多模态知识库 | Lv.2 | 已实现文本、病例影像、结构化 mask 统计表的统一检索 |

当前自选模块难度和为 `3 + 2 = 5`，满足课程要求的“总难度至少 4”。

## 二、核心功能

系统可以围绕医学图像分割主题回答问题，例如 U-Net、Mamba、MedSAM、Dice、训练策略、损失函数等。

主要特点：

- 默认完全本地运行，不依赖联网检索。
- 回答内容来自提交包内知识资产。
- 每条回答可追溯到本地知识资产来源。
- DashScope 仅作为可选增强，默认关闭。

推荐演示问题：

## 三、Lv.3 进阶动态编排

本项目已从原来的静态编排基线升级为 Lv.3 进阶动态编排。静态工作流仍保留为 baseline，但主申报模块为动态编排。

### 1. 动态编排做了什么

动态编排不是简单从 `workflows.json` 里选择一个固定流程，而是根据用户问题实时完成以下步骤：

1. 分析问题意图：普通问答、证据问答、病例分析、对比分析、模型预测演示。
2. 估计任务复杂度：是否需要文献、病例、影像、mask、表格、模型预测或恢复机制。
3. 从 `agent_registry.json` 中选择可用 Agent。
4. 生成运行时 workflow 节点。
5. 执行顺序、并行和条件节点。
6. 根据中间结果动态插入新节点。
7. 如果证据不足，触发 query rewrite retry。
8. 如果模型不可用或病例错误，触发 fallback recovery。
9. 保存完整决策轨迹和最终报告。

### 4. 动态恢复案例

典型恢复场景：

```text
Run MedSAM prediction for case NPC_001; if unavailable, recover gracefully.
```

当 MedSAM 不可用时，系统不会直接失败，而是：

1. 记录 `medsam-agent` 失败。
2. 调用 `RecoveryManager`。
3. 切换到 `ground-truth-overlay-agent`。
4. 使用真实标注 overlay 保持演示流程继续。
5. 在 `dynamic_recovery_case.json` 中保存恢复决策。

这说明系统具备可审计的 fallback recovery 能力。

## 七、功能边界说明

- 本地问答和评测默认离线运行，不抓取在线网页。
- DashScope 是可选增强，默认关闭，不影响评分证据。
- MedSAM 当前用于 2D 当前切片预测演示。
- SegRap2023 官方高精度推理需要 GPU、Docker 和官方权重环境。
- 内置 NPC 演示数据为单模态 CT；没有增强 CT 时，SegRap2023 适配器采用复制 CT 的演示兼容策略。
- 若没有官方 SegRap2023 输出，系统生成的本地 demo mask 只用于展示可视化流程，不等同于官方模型真实预测。

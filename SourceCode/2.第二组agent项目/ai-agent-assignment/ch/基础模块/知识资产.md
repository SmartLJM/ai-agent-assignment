# 1. 概述

## 1.1 知识集

> [!important]
> 本规范中使用**知识集**一词，指一个逻辑上自洽的知识整体——如一篇学术论文、一本教材、一门课程、一段百科条目、一张图片等。

**独立知识集的界定标准**（满足任一即可）：
- 有唯一标识符（DOI、URL、版本号），可独立访问
- 有独立的创作时间和作者 / 发布者
- 内容自洽，不依赖上下文即可理解

---

# 2. 分类体系

每个知识集通过 `theme`（主题）和 `keywords`（关键词）进行标注，存储于 `keywords.json`。

> [!important]
> 主题和关键词全部使用 **kebab-case** 格式。

## 2.1 主题词表

**示例**：

| 主题 | 说明 |
| --- | --- |
| `base` | 基础 / 通识 |
| `neural-network` | 神经网络 |
| `protein-corona` | 蛋白冠 |
| `genomics-sequencing` | 基因组学与测序 |
| `drug-discovery` | 药物发现 |
| `bio-imaging` | 生物影像 |
| `single-cell-omics` | 单细胞组学 |
| `climate-science` | 气候科学 |

## 2.2 关键词

`keywords` 只保留该知识集**实际涉及**的关键词，不堆砌。

| 主题 | 关键词示例 |
| --- | --- |
| `base` | `research`、`review`、`wet-experiment`、`computational-model` |
| `protein-corona` | `protein-nanoparticle-interaction`、`metal-based` |
| `genomics-sequencing` | `sequence-analysis`、`multi-omics`、`phylogenetics` |
| `drug-discovery` | `virtual-screening`、`molecular-docking`、`de-novo-design` |
| `bio-imaging` | `cell-segmentation`、`tissue-classification`、`medical-image-reconstruction` |
| `single-cell-omics` | `cell-type-annotation`、`trajectory-analysis`、`spatial-transcriptomics` |

## 2.3 标注文件格式

关键词数据保存为`keywords.json`文件，示例：

```json
{
  "base": ["review"],
  "protein-corona": ["protein-nanoparticle-interaction", "metal-based"]
}
```

# 3. 来源类型

## 3.1 类型与允许格式

| 类型 | 文件夹 | 允许格式 |
| --- | --- | --- |
| 学术论文 | `01-academic/` | PDF、MD、TXT |
| 学科教材 | `02-textbook/` | PDF、MD、TXT、EPUB |
| 课程资源 | `03-course/` | PDF、PPT、MD、Word、TXT |
| 百科资源 | `04-encyclopedia/` | PDF、MD、TXT、HTML |
| 项目工程文件 | `05-project/` | 不限（不含流数据） |
| 领域知识库 | `06-knowledge-base/` | 不限（不含流数据） |
| 技术博客 | `07-blog/` | PDF、MD、TXT、HTML |
| 领域论坛 | `08-forum/` | PDF、MD、TXT、HTML |

## 3.2 常见来源举例

| 类型     | 来源                           |
| ------ | ---------------------------- |
| 学术论文   | Nature、Science、ScienceDirect |
| 学科教材   | 正式出版教材                       |
| 课程资源   | MIT OpenCourseWare、MOOC      |
| 百科资源   | Wikipedia、百度百科               |
| 项目工程文件 | GitHub、Hugging Face          |
| 领域知识库  | UniProt、Gene Ontology、PDB    |
| 技术博客   | Medium、知乎                    |
| 领域论坛   | Reddit、Stack Overflow        |

## 3.3 采集注意事项

- 除可公开 API 访问的资源外，原始文件**必须保存到本地**
- 网页抓取时去除无关内容（广告、导航栏等）；
- 领域论坛的问答数据必须**完整保留双方问答轨迹**，不得节选或删除；涉及外链指向的信息需补全上下文；
- 所有搜集到的知识集都必须纳入知识库，且智能体可以检索到；
- 如果有搜集"项目工程文件"类型的知识集，则必须把该项目构建为MCP工具或者SKILL，智能体必须可以直接使用。

## 4.1 学术论文

  "doi": "10.18653/v1/2026.findings-eacl.62",

# 5. 存储结构

```
data/
├── 01-academic/          # 学术论文
├── 02-textbook/          # 学科教材
├── 03-course/            # 课程资源
├── 04-encyclopedia/      # 百科资源
├── 05-project/           # 项目工程文件
├── 06-knowledge-base/    # 领域知识库
├── 07-blog/              # 技术博客
└── 08-forum/             # 领域论坛
```

每个知识集在其类型文件夹下以 `{knowledge_set_id}/` 为单元组织：

```
{knowledge_set_id}/
├── content/              # 原始内容
│   ├── xxx
│   └── ...
├── keywords.json         # 主题与关键词标注
└── source.json           # 来源元数据
```

`knowledge_set_id` 为该知识集的唯一标识。

# 6. 作业要求

## 6.1 知识集数量要求

每位学生须搜集并整理**至少 20 个有效知识集**，方可计入本次作业成绩。

每个知识集须同时包含以下三个文件，缺一不可：

| 必需文件 | 说明 |
| --- | --- |
| `content/` | 原始内容（允许格式见本规范第 3.1 节） |
| `keywords.json` | 主题与关键词标注（kebab-case 格式） |
| `source.json` | 来源元数据（学术论文须含 DOI） |

> [!tip]
> 数量以实际"有效"知识集计——重复提交相同内容、不同ID指向同一来源、或缺少任一必需文件的，不计入有效数量。

## 6.2 知识集入库要求

所有搜集到的知识集**必须纳入知识库**，且智能体须能够成功检索到。入库流程须符合本规范第 3.3 节和第 5 章的结构要求。

## 6.3 项目工程文件特殊要求

若有搜集"项目工程文件"类型的知识集，则必须将该项目构建为**MCP 工具**或**SKILL**，且智能体须可以直接调用该工具，不得仅以原始代码形式堆放。

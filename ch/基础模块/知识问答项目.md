# 知识问答项目

## 概述

本项目为课程基础模块的第三部分，要求完成一个简单的知识问答系统。任务包括：
1. 根据选定的主题收集知识资产
2. 构建向量知识库，支持语义检索  
3. 实现基于检索增强生成（RAG）的智能体问答系统


## 框架推荐

### 1. Python 生态成熟框架

#### **LangChain / LangGraph**
- **定位**：Python 生态最流行的 Agent 工程平台
- **特点**：生态完善（1 亿+ 月下载量），组件丰富，灵活性极高
- **适用**：需要高度定制化、生态集成复杂的场景
- **学习成本**：中（概念较多，学习曲线较陡）
- **一句话**：如果只能学一个框架，LangChain 是最通用的选择

#### **CrewAI**
- **定位**：角色驱动的协作智能体框架
- **特点**：设计直观，将 Agent 模拟为公司员工，代码简洁
- **适用**：需要角色分工协作的场景（多智能体），但也可用于单智能体 RAG
- **学习成本**：低（20 行代码即可完成 Hello World）
- **一句话**：适合快速构建角色化的智能体工作流

### 2. 轻量级 / 新兴框架

#### **Agno**
- **定位**：轻量、简洁的 AI 应用框架
- **特点**：多模型支持，API 简洁，适合快速原型
- **适用**：教学演示、快速原型开发
- **学习成本**：低
- **一句话**：最轻量的入门选择

#### **PydanticAI**
- **定位**：基于 Pydantic 的 SDK 封装范式
- **特点**：强类型验证，开发体验接近传统 Web 开发
- **适用**：重视代码质量和类型安全的团队
- **学习成本**：低（如果你熟悉 Pydantic）
- **一句话**：TypeScript 风格的 Python AI 开发

### 3. 平台级 / 企业框架

#### **AutoGen (微软)**
- **定位**：事件驱动的多智能体系统
- **特点**：微软出品，生态支持良好，功能完整
- **适用**：复杂多智能体协作、企业级应用
- **学习成本**：中
- **一句话**：企业级多智能体协作的可靠选择

#### **Dify**
- **定位**：低代码 AI 应用平台
- **特点**：可视化编排，开箱即用，部署简单
- **适用**：不擅长编程但希望快速构建 AI 应用
- **学习成本**：极低（但定制能力有限）
- **一句话**：通过可视化界面快速搭建 RAG 应用
